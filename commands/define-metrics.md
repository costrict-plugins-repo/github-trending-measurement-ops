# /define-metrics
Set up how a program will be measured.
## Steps
1. **Choose KPIs** — Pick leading, lagging, and health metrics using `program-kpis` skill.
2. **Design the view** — Lay them out for readers using `health-dashboard` skill.
3. **Wire up research** — Coordinate the insight pipeline using `research-ops-coordination` skill.
4. **Add ops metrics** — Track delivery friction using `designops-metrics` skill.
## Output
A measurement framework: defined KPIs with baselines, a dashboard layout, and an ops-metrics plan.
Consider following up with `/report-impact` once there's data to report.
